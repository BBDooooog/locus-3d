16. 验收标准（Acceptance Criteria）

完成后应满足：

原有 GPX/FIT 导入、解析、显示功能无任何回归问题。
首页新增右侧 Terrain (DEM) 面板，可选择 .tif/.tiff 文件。
成功解析 GeoTIFF，读取高程网格（Elevation Grid）。
根据高程生成 Three.js 地形 Mesh，并加入场景。
地形可通过 OrbitControls 旋转、缩放、平移浏览。
多次导入 DEM 时，仅替换 TerrainGroup，轨迹保持不变。
未导入 DEM 时，原项目行为与当前版本完全一致。
新增代码集中在 terrain/ 模块，禁止侵入式修改已有轨迹模块。
所有新增类型、状态和渲染逻辑均遵循现有项目代码风格和组织方式。

我建议再补充一条AI 开发约束，这样大模型基本不会乱改已有代码：

Implementation Constraints

This is an incremental feature, not a refactor.
Preserve all existing GPX/FIT import and track rendering functionality.
Do not modify existing track parsing, rendering, or state management unless strictly necessary for integration.
All terrain-related logic must be implemented as an independent module (terrain/*).
The integration point should only register the terrain module into the existing Three.js scene, without changing existing rendering behavior.
